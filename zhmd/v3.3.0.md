## 摘要

**v3.3.0** 将仍残留的 ComfyUI 节点 class ID 从 Nunchaku 前缀统一为 HSWQ 前缀。量化算法与加载路径行为不变；仅节点注册 ID、类名、显示标题及相关 JS hooks 对齐。

## 更改

- **`NunchakuSaveImage`** → **`HSWQSaveImage`**
- **`NunchakuUssoewwinCheckpointLoaderSDXL`** → **`HSWQCheckpointLoaderSDXL`**
- **`NunchakuUssoewwinSDXLLoraStackV3`** → **`HSWQSDXLLoraStackV3`**（TITLE / CATEGORY 同步为 HSWQ）
- **`NunchakuZImageDiTLoader`** → **`HSWQZImageDiTLoader`**（TITLE / CATEGORY 同步为 HSWQ）
- 相关前端 hooks：`js/hswq_checkpoint_loader_sdxl_title.js`、`js/sdxl_lora_dynamic_v3.js` 的节点 ID / extension 名称改为 `hswq.*`
- INT8 / NVFP4 侧对 Checkpoint Loader 的 class 查找键同步为 `HSWQCheckpointLoaderSDXL`

## 文档

- 更新日志：`changelog.md` / `zhmd/CHANGELOG.md` → Version 3.3.0

## 操作说明

1. 将本自定义节点更新 / 重新安装到 **3.3.0**。
2. 重启 ComfyUI。
3. 旧工作流中仍使用 `NunchakuSaveImage`、`NunchakuUssoewwinCheckpointLoaderSDXL`、`NunchakuUssoewwinSDXLLoraStackV3` 等 class_type 的节点需删除后重新添加（新 ID 为 `HSWQ*`）。
